Git教程（廖雪峰的官方网站）：https://liaoxuefeng.com/books/git/time-travel/working-stage/index.html

1. 创建版本库
	1.1 创建一个空目录
		$ mkdir learngit	%创建一个文件夹
		$ cd learngit		%进入某个文件夹（或用于切换进入某个文件夹，地址开头是/c/Users/...）
		$ pwd			%显示当前目录
	1.2 创建Git可以管理的仓库
		$ git init			%初始化一个Git仓库
	1.3 把文件添加到仓库
		$ git add readme.txt					%把文件添加到仓库暂存区
		$ git commit -m "wrote a readme file" 	%将文件从暂存区提交到仓库
2. 全局配置Git所使用的邮箱和昵称
	git config --global user.email "song971222@163.com"	%配置全局提交邮箱
	git config --global user.name "LINKS"				%配置全局提交用户名
3. 查看提交历史、当前文件状态、提交新版本
	3.1查看提交历史、当前文件状态
		$ git log			%查看提交历史
		$ git status		%查看当前文件状态
		$ git diff Git教程.txt	%查看修改的内容（修改后的文件未提交仓库可用于查看修改的内容对比）
	3.2修改文件后，再次提交新版本
		$ git add .						%将修改的文件加入暂存区
		$ git commit -m "修改内容备注"		%提交新版本（把暂存区的内容提交到当前分支）
4. 版本回退
	4.1回退到某个版本状态（HEAD表示最新版本，HEAD^表示上一个版本，HEAD^~100表示往前100个版本）
		$ git reset --hard HEAD^	
（--hard参数有啥意义？--hard会回退到上个版本的已提交状态，而--soft会回退到上个版本的未提交状态，--mixed会回退到上个版本已添加但未提交的状态）
		$ git reset --hard XXXXX		
（XXX表示$git log历史记录中不同版本的版本号commit id，可以用于回到历史记录中的某个版本状态（跳转）。注意：只要记得不同版本的版本号commit id（版本号没必要写全，前几位就可以了，Git会自动去找），在Git中总是可以跳转到某个版本状态）
5. 撤销修改
	$ git checkout -- readme.txt	%把文件readme.txt在工作区的修改全部撤销（已经通过$ git add添加进暂存区的修改将不会被该指令撤销）
